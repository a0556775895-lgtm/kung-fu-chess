# Kung Fu Chess

משחק שחמט בזמן אמת: אין תורות, ושני הצדדים יכולים להזיז כלים בו-זמנית. כל תנועה נמשכת זמן בהתאם למרחק, ובזמן הזה היריב יכול להגיב, לנוע ולקפוץ.

הפרויקט תומך גם במשחק מקומי וגם במשחק מרובה משתתפים בין שני לקוחות גרפיים המחוברים לשרת WebSocket סמכותי.

## תכונות עיקריות

- **תנועה בזמן אמת** — כל הכלים יכולים לנוע בו-זמנית. משך התנועה תלוי במרחק, למעט פרש שנע במשך זמן קבוע.
- **קפיצה** — קליק ימני שולח כלי לקפיצה זמנית. בזמן הקפיצה הוא חסין לתפיסה רגילה, אך עדיין יכול להיפגע מכלי שמגיע לאותה משבצת.
- **מנוחה (cooldown)** — לאחר מהלך או קפיצה הכלי ממתין לפני שיוכל לפעול שוב.
- **הכתרת רגלי** — רגלי שמגיע לשורה האחרונה הופך למלכה.
- **ניקוד סמכותי** — השרת מחשב את הניקוד לפי הכלים שנתפסו: רגלי=1, פרש/רץ=3, צריח=5 ומלכה=9. תפיסת המלך מסיימת את המשחק.
- **יומן מהלכים** — מהלכים שהושלמו מוצגים בסימון אלגברי עם חותמת זמן.
- **Multiplayer** — שני לקוחות חולקים משחק אחד ומקבלים מהשרת את אותו מצב לוח.
- **חשבונות משתמשים** — הרשמה וכניסה עם שם משתמש וסיסמה, הנשמרים ב-SQLite בצד השרת. שם שכבר מחובר אינו יכול להתחבר שוב.
- **שמות שחקנים** — שמות השחקנים מוצגים בפאנלים לצד הניקוד במקום הכיתובים White ו-Black.
- **שרת סמכותי** — הלקוח שולח בקשות בלבד; חוקיות המהלכים, הזמן, ההתנגשויות, התפיסות והניקוד נקבעים בשרת.
- **חוקי שחמט בסיסיים** — תנועה וחסימה לפי סוג הכלי, ללא check, checkmate, castling או en passant.

## דרישות והתקנה

- Python 3.13 או גרסה תואמת.
- OpenCV, NumPy, WebSockets ו-pytest — מפורטים ב-`requirements.txt`.

מתוך תיקיית הפרויקט:

```powershell
pip install -r requirements.txt
```

## הרצת משחק מקומי

המצב המקומי מפעיל את הלוח והמנוע באותו תהליך, ללא שרת:

```powershell
python main.py
```

## הרצת משחק מרובה משתתפים

יש לפתוח שלושה חלונות PowerShell מתוך תיקיית הפרויקט.

בחלון הראשון מפעילים את השרת:

```powershell
python -m server.main
```

בחלון השני מפעילים את הלקוח הראשון:

```powershell
python -m client.main
```

בחלון השלישי מפעילים את הלקוח השני:

```powershell
python -m client.main
```

בכל לקוח בוחרים `Login` או `Register`, ואז מזינים שם משתמש וסיסמה. בהרצה הראשונה יש לרשום שני חשבונות שונים; בהרצות הבאות ניתן להתחבר אליהם. סיסמה חדשה חייבת להכיל לפחות 15 תווים. השחקן הראשון מקבל את הצבע הלבן והשחקן השני את הצבע השחור.

כדי להתחבר לשרת בכתובת אחרת:

```powershell
python -m client.main --server ws://127.0.0.1:8765
```

## שליטה

- **קליק שמאלי** — בחירת כלי, ולאחר מכן בחירת משבצת היעד.
- **קליק ימני על כלי** — קפיצה.
- **Esc** — סגירת חלון המשחק.

## תהליך ההתחברות

לפני שהלקוח יכול לשלוח פקודות משחק הוא משלים handshake מסודר:

```text
REGISTER או LOGIN
  ↓
AuthService + SQLite
  ↓
AUTH_OK
  ↓
JOIN עם הקונפיג המבוקש
  ↓
CONFIG_ACCEPTED או CONFIG_OVERRIDDEN
  ↓
STATE ראשוני
```

לאחר מכן הלקוח שולח פקודות `MOVE` ו-`JUMP`. השרת מאמת הרשאות וחוקיות ומפיץ בחזרה הודעות `STATE`, ‏`EVENT`, ‏`OK` או `ERR`.

## ארכיטקטורת Multiplayer

```text
קלט עכבר
   ↓
RemoteGameEngineProxy
   ↓
NetworkClient
   ↓ WebSocket
GameServer
   ↓
GameAdmission / GameController
   ↓
Match
   ↓
GameEngine
   ↓
EventBus + ServerBroadcaster
   ↓ STATE / EVENT
NetworkClient → Proxy → View
```

הלקוח אינו משנה את מצב המשחק בעצמו. הוא מציג snapshots מהשרת ושולח בקשות לפעולות. רק השרת מקדם את שעון המשחק ומכריע תנועה, הגעה, תפיסה, קפיצה, מנוחה וניקוד.

במצב המקומי `main.py` מחבר את `DisplayManager` ישירות ל-`GameEngine`. במצב מרובה משתתפים `client.main` מחבר אותו ל-`RemoteGameEngineProxy`, ולכן אותה תצוגה יכולה לעבוד מול מנוע מקומי או מול שרת.

## בדיקות

הרצת כל בדיקות היחידה והאינטגרציה:

```powershell
python -m pytest -q
```

הרצה עם דוח כיסוי:

```powershell
./run_tests.ps1
```

בנקודת העדכון האחרונה עברו 273 בדיקות. הבדיקות כוללות את חוקי המשחק, serialization, הרשאות, שני לקוחות WebSocket אמיתיים, הרשמה וכניסה מול SQLite, סנכרון מצב והצגת שמות השחקנים.
